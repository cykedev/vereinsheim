**TL;DR** Der Memory-Graph ist ab v3 ein **Obsidian-kompatibler `vault/` aus Markdown-Notes**, die **zugleich** der Graph sind — kein Build-Schritt, keine separate Store-Datei: `vault-loader.mjs` liest die Notes direkt, der Agent editiert sie **live**. Root- **und** App-Doku werden in **einen** Vault konsolidiert. Die BM25/EN+DE-Such-Engine aus [[adr-022]] bleibt erhalten; nur ihre Quelle wechselt von `knowledge-graph.json` auf `vault/`.

## Kontext

Der Memory-Graph war seit [[adr-022]] ein **gebauter, relationenreicher Doku-Index**: `.claude/build-graph.mjs` erzeugte aus drei eingecheckten Quellen (`docs/decisions.md`, `.claude/graph-projection.mjs`, `.claude/graph-captured.mjs`) deterministisch den Store `.claude/knowledge-graph.json`, gelesen über Fragment-Pointer (`→ datei#slug`) via `doc.mjs`. Das löste die Selbstaktualisierungs- und Token-Probleme von [[adr-021]], hinterließ aber drei eigene Schwächen:

- **Build-Reibung + Zwei-Wahrheiten-Fläche.** Wissen lebte in den Docs *und*, projiziert, im Manifest (`graph-projection.mjs`); jede inhaltliche Änderung brauchte einen Rebuild und Disziplin, die Projektion nachzuziehen (`/sync-graph`, modellgetrieben). Der Store war ein Artefakt, das nie von Hand editiert werden durfte — eine dauerhafte Fehlerquelle.
- **Kein direktes Editieren.** Neues Wissen musste in eine Quelle geschrieben + gebaut + committet werden; ein Live-`mcp__memory__`-Write wäre vom nächsten Rebuild überschrieben worden. Der Umweg über die Quelle war korrekt, aber umständlich.
- **Doku in zwei getrennten Schichten.** Root-`docs/` (universell) und `apps/<app>/docs/*` (app-lokal, on-demand via [[adr-019]]) waren separate Korpora; der Index spannte zwar beide, aber das Wissen selbst lag verstreut, mit unterschiedlichen Lade-/Suchpfaden.

Parallel hat sich in einem Schwesterprojekt (`cs-knowledge-graph`) das Muster „Markdown-**ist**-Graph" bewährt: eine Menge von Notes mit typisierten Frontmatter-Kanten, die gleichzeitig ein browsebarer Obsidian-Vault und der von einem headless MCP-Server servierte Korpus ist — ohne Build-Schritt.

## Entscheidung

Das Wissenssystem wird auf einen **Obsidian-kompatiblen Vault-Graph** umgestellt (Blueprint-Herkunft: `basic-harness` v3.1). Zwei zusammengehörige Teilentscheidungen:

**(a) Engine — der Vault *ist* der Graph (löst [[adr-022]] ab).**
- `vault/` enthält eine Markdown-Note pro Entity; der Dateiname (kebab, ohne `.md`) **ist** die Node-Id, typisierte Frontmatter-Listen (`part_of`/`governed_by`/`documented_in`/`supersedes`/…) sind die Kanten. Ordner = Topic, jedes Topic hat einen MOC-Hub (`type: guide`).
- **Kein Build, keine Store-Datei.** `.claude/vault-loader.mjs` liest die Notes direkt in den Graph; der Agent editiert die Note **live** (Änderung sofort wirksam). `.claude/vault-lint.mjs` validiert Schema + Anker + Orphans (im `graph-sync`-Stop-Hook, ersetzt `build-graph.mjs`).
- **Retrieval:** `mcp__memory__search_nodes` (BM25) → `document_map` → `section_read`, plus `open_nodes`/`backlinks`/`traverse`. `doc.mjs` bleibt als Fragment-Reader erhalten.
- **Die Such-Engine aus [[adr-022]] bleibt unverändert** (`search-index.mjs`, BM25 + EN/DE-Stopwords/Synonyme/Prefix); nur ihre Datenquelle wechselt vom gebauten Store auf den Vault. Deshalb ist dies eine Ablösung der *Speicher-/Build*-Entscheidung von [[adr-022]], **nicht** der Such-Qualität.
- **Obsidian-Kompatibilität ist eine Eigenschaft des Formats, nicht der Laufzeit:** nichts in der Harness braucht Obsidian; flache `[[wikilink]]`-Kanten + Heading-Text-Anker machen den Vault nur *nebenbei* in Obsidian browsebar (Graph-View, Backlinks).

**(b) Konsolidierung — ein einziger Wissensspeicher (schärft [[adr-019]]).**
- Root-`docs/` **und** die App-Schicht `apps/<app>/docs/*` werden vollständig in **einen** Vault überführt (`vault/apps/ringwerk/`, `vault/apps/treffsicher/`). Die alten `docs/`-Bäume entfallen; `apps/*/docs/superpowers/` (erledigte Alt-Pläne, [[adr-020]]) bleiben als Historie liegen.
- Die **CLAUDE.md-Governance-Hierarchie** (Root → App → package) aus [[adr-019]] **bleibt** — sie trägt Regeln/Hinweise für den Agenten. Geändert wird nur die **Doku-Schicht**: Wissen wird nicht mehr per `@import`/on-demand aus `apps/*/docs/` geladen, sondern zentral im Vault gesucht. App-`CLAUDE.md` verweist auf den Vault statt auf `apps/*/docs/`.

## Alternativen

- **Gebauter Index beibehalten ([[adr-022]], Status quo).** Verworfen: Build-Reibung, Zwei-Wahrheiten-Fläche (Docs + Manifest), kein Live-Editieren, „Store nie von Hand anfassen" als Dauer-Stolperstein. Der Vault beseitigt den Build ganz.
- **App-Doku app-lokal lassen, Vault indexiert sie nur** (schlanke Verweis-Notes, Volltext bleibt in `apps/*/docs/`). Verworfen: hielte zwei Suchpfade + zwei Wahrheiten aufrecht; der Wunsch war **ein** Speicher, in dem Regeln/Architektur/Entwicklungsstile vollständig auffindbar sind. Der Preis (Schärfung von [[adr-019]]s Doku-Schicht) ist bewusst akzeptiert.
- **Vektor-RAG über Doc-Chunks.** Schon in [[adr-022]] verworfen (externe Infra, weniger kuratiert/präzise als ein relationenreicher Graph; für dieses Volumen überdimensioniert) — gilt weiter.
- **Obsidian „Local REST API"-Plugin als Laufzeit-Abhängigkeit.** Verworfen: die Harness bleibt clone-and-go, headless, zero-dependency; Obsidian-Kompatibilität ist Format-, keine Laufzeit-Eigenschaft.

## Folgen

- **Direktes Editieren:** neues Wissen = Note anlegen/ändern (kein Build, kein Rebuild-Commit-Zyklus). `/sync-graph` validiert/pflegt nur noch (vault-lint + Cross-Refs), baut nicht. `/consolidate-lessons` REMEMBER schreibt eine `incident`-Note live statt in eine Quelle + Rebuild.
- **Stop-Hook:** `graph-sync.mjs` ruft `vault-lint.mjs` (Schema/Anker/Orphans) statt `build-graph.mjs`; blockt das Turn-Ende bei invalidem Vault.
- **Entfällt:** `build-graph.mjs`, `graph-projection.mjs`, `graph-captured.mjs`, `knowledge-graph.json(.search.json)`, der ganze Root-`docs/`-Baum und die migrierten `apps/*/docs/*.md`. **Bleibt:** `doc.mjs` (Reader), `search-index.mjs`/`search-synonyms.mjs` (Engine + DE-Synonyme), `graph-store.mjs`.
- **[[adr-016]]** (Graph = Layer 3, **nicht** Struktur-Autorität) bleibt gewahrt: der Vault macht Doku auffindbar, ist nicht ihre alleinige Wahrheit — CodeGraph bleibt die Ground Truth über den Code.
- **Migrations-Vollständigkeit:** die Umstellung überführt Root- + App-Doku **verlustfrei** (Volltext) und alle 116 v2-Entities als Notes; ein Completeness-Gate sichert das vor dem Löschen der Altquellen ab.
- **Determinismus-Grenze entfällt:** die in [[adr-022]] beschriebene Grenze (Docs→Manifest modellgetrieben, Build deterministisch) ist gegenstandslos — es gibt kein Manifest und keinen Build mehr.
